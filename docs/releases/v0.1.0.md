# Release `v0.1.0`

## Resumo

Primeiro marco estável de governança e pipeline do projeto.

## Inclui

- política oficial de branches;
- convenção de deploy para backend e frontend/admin;
- CI ajustado para ignorar `admin-deploy`;
- validação do frontend no publish workflow;
- política de tags e releases;
- alinhamento do Heroku com `main` e `admin-deploy`.

## Impacto operacional

- `main` publica o backend;
- `admin-deploy` publica o frontend administrativo;
- o frontend passa a ser validado antes da publicação da branch de
  distribuição;
- o backend deixa de rodar CI em `admin-deploy`.

## Observações

- Este marco não representa ainda uma versão final do produto.
- Ele consolida a base operacional para evolução segura do repositório.
- A tag correspondente é `v0.1.0`.
